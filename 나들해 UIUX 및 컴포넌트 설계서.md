# 전주 피크닉 & 반나절 코스 서비스: 프론트엔드 UI/UX 및 컴포넌트 설계서

## 1. 디자인 컨셉 및 톤앤매너

- **핵심 키워드:** 감성적인(Emotional), 직관적인(Intuitive), 마법 같은(Magical - AI 추천 체감)
    
- **메인 컬러:** 전주의 맑은 하늘을 상징하는 **스카이 블루(#87CEEB)**와 한옥마을의 따뜻함을 담은 **우드 베이지(#F5F5DC)**.
    
- **UI 라이브러리:** `Magic UI` (애니메이션 중심), `Tailwind CSS` (레이아웃 및 유틸리티)
    
- **데이터 흐름(현재 상태):** 백엔드가 아직 개발 중이므로, 사전에 협의된 API 명세서를 바탕으로 프론트엔드 내부에 가짜 데이터(Mock Data)를 만들어 화면의 흐름과 UI를 먼저 완성합니다.
    

## 2. 화면 흐름(User Flow)에 따른 UI 레이아웃 설계

사용자가 웹사이트에 들어와서 스크롤을 내리며 경험하게 될 흐름 순서대로 화면을 설계합니다.

### Flow 1: 진입 및 첫인상 (Hero Section)

사용자가 접속하자마자 가장 먼저 보는 영역입니다. 오늘의 날씨와 피크닉 가기 좋은 정도를 시각적으로 강렬하게 전달해야 합니다.

- **사용자 경험:** "우와, 오늘 전주 날씨 진짜 좋구나! 당장 나가고 싶다."
    
- **배치될 Magic UI 컴포넌트:**
    
    1. **`Particles` 또는 `Meteors` (배경 애니메이션):** * 날씨가 맑은 날에는 `Particles` 컴포넌트로 화면 뒤에 벚꽃 잎이나 민들레 홀씨가 흩날리는 듯한 효과를 줍니다.
        
        - 흐리거나 밤인 경우 `Meteors`를 활용해 감성적인 밤하늘을 연출합니다.
            
    2. **`Word Pull Up` 또는 `Blur In` (메인 타이틀):** * "오늘 전주는 피크닉 가기 완벽한 날이에요!"라는 텍스트가 부드럽게 떠오르며 시선을 사로잡게 합니다.
        
    3. **`Number Ticker` (피크닉 지수):** * 중앙에 크게 배치된 동그란 위젯 안에 피크닉 지수 "95"점이 0부터 95까지 차르르륵 올라가는(Counting) 애니메이션을 주어 시각적 쾌감을 줍니다.
        

### Flow 2: 과거 데이터 기반 인사이트 (Statistics Section)

스크롤을 살짝 내리면 나오는 영역입니다. 단순한 오늘 날씨를 넘어, 통계 데이터를 예쁘게 보여주어 서비스의 신뢰도와 흥미를 높입니다.

- **사용자 경험:** "오, 작년 이맘때는 이랬구나. 통계가 한눈에 들어와서 재밌네."
    
- **배치될 Magic UI 컴포넌트:**
    
    1. **`Bento Grid` (핵심 레이아웃):**
        
        - 화면을 여러 크기의 둥근 사각형(카드)으로 분할하는 Bento Grid를 사용합니다.
            
        - 큰 카드에는 "5월 전주 날씨 요약 그래프", 작은 카드들에는 "가장 피크닉 하기 좋았던 요일: 토요일", "덕진공원 평균 체감 온도" 등을 각각 배치하여 정보의 위계를 나눕니다.
            
    2. **`Marquee` (무한 스크롤 띠):**
        
        - Bento Grid 하단에 텍스트가 왼쪽으로 계속 흘러가는 `Marquee`를 배치합니다.
            
        - 내용: _"지금 전주 시민들이 많이 찾는 스팟: #세병호 #덕진공원 #객리단길 #전라감영 ..."_ (실시간 트렌드 같은 느낌을 줍니다).
            

### Flow 3: 반나절 코스 생성 (AI Action Section)

이 서비스의 핵심인 시간대 입력 및 AI 코스 추천 영역입니다. 사용자가 직접 조작하고 '기다리는' 흐름이 포함됩니다.

- **사용자 경험:** "오늘 오후에 뭐 할지 물어봐야지. (입력 후 대기) 오, 내 일정에 딱 맞춰서 짜주네!"
    
- **배치될 Magic UI 컴포넌트:**
    
    1. **`Shimmer Button` (코스 생성 버튼):**
        
        - 시간을 선택하는 UI(예: 13:00 ~ 18:00) 아래에 배치합니다. 버튼 표면에 빛이 스윽 지나가는 애니메이션을 주어 "이 버튼을 누르면 추천이 시작된다"는 특별한 느낌을 강조합니다.
            
    2. **`Animated Circular ProgressBar` (로딩 화면):**
        
        - 서버에 데이터를 요청하고 기다리는 동안 렌더링 됩니다. "날씨 흐름을 분석하여 전주 코스를 구성 중입니다..."라는 텍스트와 함께 예쁜 프로그레스 바를 보여주어 기다림을 지루하지 않게 합니다.
            

### Flow 4: 결과 확인 (Result Timeline Section)

서버에서 응답받은 시간대별 날씨 변화와 동선을 보여주는 영역입니다.

- **사용자 경험:** "1시엔 야외, 4시엔 실내... 날씨 흐름에 딱 맞춘 완벽한 동선이다!"
    
- **배치될 Magic UI 컴포넌트:**
    
    1. **`Shine Border` 또는 `Border Beam` (결과 카드 테두리):**
        
        - 추천된 결과가 담긴 큰 카드 컴포넌트의 테두리를 따라 빛이 빙글빙글 도는 효과를 줍니다.
            
    2. **`Animated List` (타임라인 리스트):**
        
        - 시간대별 코스(예: 1시 덕진공원 -> 4시 근처 카페)가 위에서 아래로 하나씩 팝업 되며 나타나도록 구성합니다.
            

## 3. 백엔드와의 병렬 개발 및 API 연동 흐름 (Integration Flow)

프론트엔드는 백엔드 개발이 끝날 때까지 기다리지 않습니다. 양측이 합의한 'API 명세서'를 바탕으로 독립적으로 화면을 개발한 뒤, 마지막에 결합하는 흐름을 따릅니다.

### 1단계: API 명세서(Contract) 확인 및 Mocking 준비

- 백엔드와 사전 협의된 노션(Notion)이나 스웨거(Swagger) 문서를 확인합니다.
    
- 명세서에 적힌 응답(Response) JSON 구조를 똑같이 모방하여, 프론트엔드 프로젝트 내부에 **가짜 데이터(Mock Data)** 파일을 만듭니다. (예: `mockWeatherData.json`)
    
- 조금 더 고도화한다면, MSW(Mock Service Worker) 같은 라이브러리를 사용하여 브라우저 단에서 가짜 API 서버가 돌아가도록 세팅합니다.
    

### 2단계: UI 로직 및 상태 관리 완벽 구현 (독립 개발)

- 백엔드가 없어도 가짜 데이터를 프론트엔드의 `useState`에 담아 화면을 그립니다.
    
- 애니메이션 타이밍, 로딩(Loading) 스피너 처리, 에러(Error) 발생 시 보여줄 팝업 등 사용자가 경험할 모든 시각적 흐름을 완성해 둡니다.
    

### 3단계: 실제 API 주소 교체 및 결합 (Integration)

- 백엔드 API 서버가 배포되면, 기존에 가짜 데이터를 가리키던 URL(예: `/mocks/api/weather`)을 **실제 백엔드 주소(예: `https://api.nadeulhae.com/weather`)로 일괄 교체**합니다.
    
- **[주의] CORS 에러 대응:** 이때 브라우저에서 'CORS 정책에 의해 차단되었습니다'라는 빨간색 에러가 십중팔구 발생합니다. 당황하지 않고 백엔드 개발자에게 "프론트엔드 주소(localhost:3000)를 허용(Allow Origin)해 주세요"라고 요청합니다.
    

### 4단계: 데이터 정합성 QA (최종 확인)

- 통신이 뚫렸다면, 이제 명세서대로 데이터가 정확히 넘어오는지 확인합니다.
    
- 백엔드에서 `picnicScore`를 숫자로 주기로 했는데 문자열 `"95"`로 넘어와서 화면의 카운팅 애니메이션이 고장 나지 않는지, 변수명에 오타(예: `userName` vs `username`)가 없는지 꼼꼼히 체크하며 버그를 잡습니다. 이 과정이 끝나면 서비스 배포 준비가 완료됩니다.
